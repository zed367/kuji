# vibe_kuji — 경품 추첨 게임(쿠지) 태블릿 프로토타입 핸드오프

> 이 문서 + 이 프로젝트 폴더(zip) 하나만 있으면 다른 환경의 Claude Code에서도 이어서 작업할 수 있습니다.

---

## 1. 개요

- **무엇**: 팬딩(Fanding)의 "경품 추첨 게임"(카드팩 매대 뽑기, 일명 쿠지) 프로토타입
- **대상 기기**: 갤럭시 탭 S6 Lite **가로(Landscape)** 기준 (10.4형, 2000×1200 = 5:3, 논리 해상도 약 1200×720)
- **기술 스택**: Vite + Vanilla JS + [`motion`](https://motion.dev) npm 패키지. 별도 프레임워크(React 등) 없음
- **범위**: 실제 유저가 보는 화면만 (관리자 페이지는 범위 밖, 추후 별도 작업)

## 2. 실행 방법

```bash
npm install
npm run dev
```

터미널에 안내되는 주소(`http://localhost:5173` 등)를 브라우저로 열면 됩니다. 가로 1200×720 근처 창 크기에서 보는 것을 권장합니다(그 기준으로 튜닝됨).

## 4. 화면 구성

- **좌측**: 상단 배너(폭 전체) + 카드팩 매대(평평한 3줄×5칸=15개 그리드, 계단식 원근감 없이 전부 같은 크기로 매대 위에 나란히 놓인 형태)
- **우측**: 1/5/10뽑기 탭, 안내 문구, 등급별 재고·확률 패널

레이아웃은 `fitStandSize()`(main.js)가 `.stand-outer`의 실제 렌더 크기를 측정해서 카드팩 한 장의 너비(`--pack-w` CSS 변수)를 역산합니다. 스크롤 없이 뷰포트에 딱 맞추기 위함이며, 창 크기가 바뀌면 자동 재계산됩니다.

### 카드팩 이미지 (매대에 진열되는 이미지)

- `src/main.js` 상단의 `PACK_IMAGES` 배열에 등록된 이미지들 중 매대 칸마다 무작위로 하나씩 골라 진열됩니다. 선택된 팩은 개봉 스테이지까지 같은 이미지로 이어집니다.
- 이미지 추가 방법: 파일을 `public/packs/`에 넣고 `PACK_IMAGES` 배열에 경로만 추가.
- 이미지 규격: **가로:세로 = 386:751 비율**로 크롭 (`PACK_ASPECT` 상수, main.js). 투명 배경 컷아웃(진짜 팩 실루엣처럼 보임)과 불투명 사각 이미지(카드처럼 보임) 둘 다 지원. 비율이 다르면 매대 그리드 크기 계산과 개봉 스테이지 크기(`.unbox-pack-bottom`, style.css)가 어긋날 수 있음.
- 현재 등록된 `pack-2.webp`~`pack-6.webp`는 **극장판 주술회전(呪術廻戦) 캐릭터 포스터를 크롭한 라이선스 이미지**입니다 — 자세한 내용은 8장 TODO 참고.

## 5. 핵심 인터랙션 (순서대로)

1. 매대에서 카드팩 탭 → 확대 피드백 애니메이션 → 개봉 스테이지(오버레이)로 전환
2. 팩 상단 절취선(점선)을 손잡이로 슬라이스(드래그, Pointer Events 기반) → 90% 이상 밀면 개봉 인정, 못 미치고 놓으면 스냅백
3. 절취선 위쪽(장식용 헤더)이 뜯겨 날아가며 개봉 — **카드팩 이미지 자체는 이 뜯기는 부분과 무관하게 절취선 아래(`.unbox-pack-bottom`)에 통째로 온전히 표시**되므로, 어떤 이미지를 넣어도 캐릭터 얼굴 등이 잘려나가는 일이 없음
4. 결과 카드가 나오기 직전, 카드팩 전체가 페이드아웃되며 사라짐 (카드가 더 크게 보일 공간 확보용, 1/5/10뽑기 공통)
5. 이후 결과 카드 연출은 **뽑은 장수에 따라 둘 중 하나**로 갈림 (`cardActionForCount()`, main.js) — 6장 참고
6. 전부 공개되면 결과 요약 + 닫기 버튼 노출, 우측 재고/확률 패널 갱신
7. **"새 카드팩" 리로드 버튼**: 매대 진열만 새로고침되는 연출 — 실제 재고/확률에는 영향 없는 시각 전용 기능 (이 점을 오해 없도록 유지할 것)

## 6. 결과 카드 연출 - card_action1 / card_action2

결과 카드를 어떤 방식으로 보여줄지는 두 가지로 구현되어 있고, `cardActionForCount(count)`(main.js) 한 곳에서 장수별로 어떤 액션을 쓸지 정합니다. 함수만 바꿔 끼우면 되므로 조합을 자유롭게 실험 가능합니다 (예: `count === 1 ? cardAction1 : cardAction2`가 현재 기본값).

- **card_action1** (`main.js`, 현재 **1뽑기**에 사용) — 뽑은 카드가 뒷면 상태로 흩뿌려진 배치로 한 번에 등장하고, 사용자가 한 장씩 탭해서 뒤집어 확인하거나 우상단 "전체 공개"로 한 번에 확인. 장수가 적을 때 어울림.
- **card_action2** (`main.js`, 현재 **5뽑기/10뽑기**에 사용) — 카드 전부가 한 장 밑에 다음 장이 살짝 겹쳐 쌓인 "스택"으로 등장. **맨 위 첫 장만 뒷면(미스터리)** 이고, 그 아래 깔린 나머지 카드들은 이미 앞면(결과)이 뒤집힌 채로 스택 속에 숨어있음 — 첫 장을 위로 슬라이스해서 넘기는 순간부터는 넘길 때마다 바로바로 다음 결과가 튀어나오는 리듬감("쪼는 맛")을 냄. 스택을 다 넘기면 card_action1과 같은 방식으로 전체 카드가 한 번에 흩뿌려진 배치로 노출됨. 스와이프 임계값은 `SWIPE_DISMISS_DY`(70px, main.js).

두 액션 모두 결과 확정 후 마무리 처리(제목/버튼/요약 노출)는 공통 함수 `finishReveal(overlay, results)`를 공유합니다.

카드 크기는 128×180px(`.card-slot`, style.css) — 개봉 시 카드팩이 사라지면서 확보된 공간에 맞춰 기존(84×118)보다 키운 값입니다.
